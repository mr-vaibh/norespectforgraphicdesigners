# No Respect for Graphic Designers

> A brutal satirical tribute to frontend developers — roasting graphic designers, no-code tools, and everyone who confuses aesthetics with usability.

---

## What's inside

| Section | What it does |
|---|---|
| **Justification Generator** | Generates designer BS quotes, excuses, and $15k invoices |
| **Dribbble Hall of Shame** | Award-winning designs with live WCAG crime severity scores |
| **Designer Bingo** | Fully playable 5×5 bingo card. You'll win fast. |
| **Figma vs. Reality** | Side-by-side of what was designed vs. what was actually built |
| **A Day in the Life** | Step-through timeline of a designer's 8 billable hours |
| **Color Palette Crime Scene** | Trendy palettes auto-tested against WCAG — they all fail |
| **CSS Art Gallery** | Live CSS demos. A developer built these. No Adobe products harmed. |
| **Lovable.dev Honest Review** | An honest look at no-code "development" |
| **Things Designers Think Are Hard** | CSS solutions to problems designers charge weeks to "solve" |
| **Can a Dev Do This?** | Live stopwatch challenge vs. designer time estimates |
| **$15k UX Audit** | Enter any URL. Get a fabricated crisis. Receive an invoice. |
| **The Manifesto** | 8 principles. The final word. |

---

## Stack

- **Vite + React** — no overhead, no excuses
- **Tailwind CSS v4** — utility-first, zero config
- **Framer Motion** — animations that don't require After Effects
- **Zero backend** — it's a frontend site, built by a frontend dev

---

## Run locally

```bash
npm install
npm run dev
```

## The point

Design matters. Bad design defended with buzzwords doesn't.  
Usability is not optional. Accessibility is not optional.  
Delivering on time is not optional.

Do better. Or get out of the way and let the developers handle it.
